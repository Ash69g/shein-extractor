# shein-extractor

تطبيق سطح مكتب لاستخراج منتجات روابط سلة SHEIN وعرضها وحفظ سجل التحليلات.

## التشغيل

```powershell
.\.venv\Scripts\Activate.ps1
python app.py
```

يمكن كذلك تثبيت المشروع محليًا بوضع التطوير وتشغيل نقطة الدخول الجديدة:

```powershell
python -m pip install -e .
shein-extractor
```

بقي الأمر `python app.py` مدعومًا بالكامل للتوافق. تفاصيل توزيع الطبقات واتجاه الاعتماديات موثقة في `ARCHITECTURE.md`.

## تصدير PDF

- يبدأ إنشاء PDF تلقائيًا بعد اكتمال التحليل وانتهاء تجهيز صور المنتجات.
- يعيد البرنامج محاولة تحميل كل صورة فاشلة حتى 10 مرات.
- إذا استنفدت الصورة جميع المحاولات، يبقى المنتج داخل التقرير وتظهر مكانها عبارة `الصورة غير متوفرة`.
- يصدّر التقرير جميع منتجات التحليل، حتى عند تفعيل بحث يخفي بعض صفوف الجدول.
- يتضمن التقرير بيانات العميل والطلب والرابط والإحصائيات وجدول المنتجات والصور.
- يقتصر اسم المنتج داخل التقرير على أول 100 حرف.
- يُحفظ التقرير تلقائيًا داخل `exports/` باسم مطابق لاسم ملف JSON.
- إذا كان التقرير موجودًا مسبقًا، لا ينشئ البرنامج نسخة أخرى؛ يفتحه زر `تصدير البيانات` مباشرة.
- يمكن إنشاء أو فتح تقرير لأي تحليل قديم بعد اختياره من السجل.
- يستخدم التقرير `A4 Portrait`، ويكرر رأس الجدول في كل صفحة دون تقسيم صف المنتج بين صفحتين.

يعرض شريط الحالة تقدم تحميل الصور ومحاولات الإعادة وإنشاء صفحات PDF. يظهر تأكيد الحفظ كإشعار مؤقت غير مانع بدل نافذة حوار.

## التحقق المحلي

بعد تفعيل البيئة الافتراضية يمكن فحص المشروع كاملًا بالأوامر التالية:

```powershell
python -m pip install -e ".[dev]"
ruff check .
pytest -q
python -m compileall -q src
```

تشمل الاختبارات تحليل الفاتورة، والتحقق من الرابط، وتطبيع جميع حالات المنتجات، وعقد `SKU`، والحفظ والتحميل، وإغلاق Chromium عند النجاح والفشل، وإنشاء PDF، وفتح وإغلاق واجهة Qt.

## تشغيل API محليًا

ثبّت اعتماديات الخادم، وعيّن مفتاحًا سريًا، ثم شغّل `uvicorn`:

```powershell
python -m pip install -e ".[server,dev]"
$env:SHEIN_API_KEY = (python -c "import secrets; print(secrets.token_urlsafe(32))")
python -m uvicorn shein_extractor.presentation.api.bootstrap:app --host 127.0.0.1 --port 8000
```

نقاط الخدمة الحالية:

- `GET /health/live`: فحص أن عملية API تعمل.
- `GET /health/ready`: فحص المفتاح ومجلدي JSON وPDF.
- `POST /v1/process`: يستقبل الفاتورة أو الرابط ويعيد ملف PDF مباشرة، ويتطلب الترويسة `X-API-Key`.
- `GET /docs`: توثيق OpenAPI التفاعلي.

مثال جسم الطلب:

```json
{
  "raw_input": "نص الفاتورة أو رابط SHEIN",
  "customer_name": null,
  "order_number": null,
  "timeout_seconds": 60
}
```

لا تحفظ قيمة `SHEIN_API_KEY` الحقيقية داخل Git. يحتوي `.env.example` أسماء متغيرات البيئة المطلوبة فقط. لم يضف طابور `FIFO` أو تكامل `n8n` وTelegram بعد؛ وهما المرحلة التالية.
